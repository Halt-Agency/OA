KEY DECISIONS TO MAKE
═══════════════════════════════════════════════════════════════════════════════

DECISION 1: EVENTS
──────────────────

Ask: "How many events do you run per year and how large are they?"

If 1-2 small events → Recommend: TrackerRMS Tags Only
If 3+ events or 50+ people → Recommend: WordPress Attendees CPT
If want contact database → Recommend: Hybrid (Both)

Client Choice: ______________________________________________


DECISION 2: NEWSLETTERS
───────────────────────

Ask: "How many newsletter signups per week?"

If under 10/week → Recommend: TrackerRMS (manual)
If over 10/week → Recommend: Mailchimp (automatic)

Ask: "Budget for Mailchimp (~£20/month)?" Yes / No

Client Choice: ______________________________________________


DECISION 3: DATA CLEANUP
─────────────────────────

Ask: "Someone available for quarterly manual review of old data?"

If Yes → Manual cleanup (no extra dev)
If No → Automated cleanup (1-2 days dev)

Client Choice: ______________________________________________

OBJECTION HANDLING
═══════════════════════════════════════════════════════════════════════════════

"This seems complicated"
→ "The backend is complex, but for your team it's simple - forms just work"

"What if someone wants their data deleted?"
→ "We have a clear process for both WordPress and TrackerRMS deletion"

"Can we change things later?"
→ "Yes! We can add features as your needs grow"

"What about Send Lists / Newsletter automation?"
→ "TrackerRMS API doesn't support this, so we recommend Mailchimp"

"How do we know it won't create duplicates?"
→ "System searches by email before creating - already tested and working"
